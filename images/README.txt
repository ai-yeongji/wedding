[이미지 폴더 안내]
====================

이 폴더에 청첩장에서 사용할 이미지 파일을 넣으세요.
파일명과 경로는 js/config.js의 설정과 반드시 일치해야 합니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
필수 이미지
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ main_visual.jpg (커버 대표 이미지)
  - config.js → CONFIG.mainVisual 경로에 대응
  - 권장 비율: 세로형 (9:16 또는 3:4)
  - 권장 해상도: 최소 860×1148px (2x retina 기준)
  - 권장 용량: 2MB 이하 (로딩 속도 최적화)
  - Open Graph 이미지로도 활용 (index.html head의 og:image 태그 URL 교체 필요)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
갤러리 이미지 (기본 6장)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ g1.jpg ~ g6.jpg
  - config.js → CONFIG.gallery 배열에 대응
  - 권장 비율: 정방형(1:1) 또는 세로형(3:4) — gallery.js 그리드 레이아웃에 맞춤
  - 권장 해상도: 600×600px 이상 (썸네일) / 1200px 이상 (라이트박스)
  - 권장 용량: 1MB 이하/장
  - 파일 수 변경 시: config.js의 CONFIG.gallery 배열도 함께 수정하세요

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
파일명 변경 방법
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

이미지 파일명을 바꾸려면 js/config.js를 열어
CONFIG.mainVisual 또는 CONFIG.gallery 배열의 경로를 함께 수정하세요.

예시 (config.js):
  mainVisual: 'images/couple_photo.jpg',   ← 파일명 변경
  gallery: [
    'images/photo_01.jpg',
    'images/photo_02.jpg',
    ...
  ],

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
이미지 최적화 권장 사항
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- 포맷: JPEG(사진) 또는 WebP(용량 최적화 시)
- 압축: squoosh.app 또는 tinyjpg.com 사용 권장
- WebP 지원: css/style.css의 .img-placeholder background-color가
  이미지 로딩 실패 시 빈 영역 깨짐을 방지하는 fallback입니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
raw_images 폴더 안내
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

images/raw_images/ 에 원본 사진 JPEG 파일이 있습니다.
이 파일들 중에서 선택하여 아래와 같이 복사/이름 변경하세요:

  1) 커버 대표 이미지로 사용할 1장 → main_visual.jpg 로 복사
  2) 갤러리에 사용할 최대 6장 → g1.jpg ~ g6.jpg 로 복사
     (6장 이상 사용하려면 config.js의 CONFIG.gallery 배열에 경로 추가)

이미지 최적화 후 images/ 폴더 루트에 배치하는 것을 권장합니다.
